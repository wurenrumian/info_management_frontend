# 学院事务小程序 CSS 基础框架与样式演示设计（人大场景）

## 1. 背景与目标

基于当前项目定位（学院事务通知与办理，学生端为主，微信小程序优先），样式系统需同时满足：

- 体现中国人民大学学院场景的正式感与可信度
- 优先保障学生端信息可读性与操作效率
- 支撑后续模块并行开发（通知、审批、知识库、党团流程）

设计结论：采用 **“学生易用性优先的人大轻正式风格”**。

## 2. 范围与非目标

### 2.1 本次范围

1. 建立全局样式基础框架（Design Tokens + 基础样式规则）
2. 新增一个“样式演示页”（style showcase）用于集中预览视觉规范
3. 提供移动端优先、兼容 H5 的布局与排版基线

### 2.2 非目标

1. 不在本次完成业务页面（通知详情、审批详情等）的完整重构
2. 不引入复杂主题切换（如深色模式）
3. 不一次性建立大型原子类体系

## 3. 视觉方向

### 3.1 风格关键词

- 正式但不压迫
- 清晰优先
- 学院感
- 温和克制
- 适合手机阅读

### 3.2 色彩策略

- 主色使用“人大红”语义色，限制大面积铺色，更多用于强调与关键操作
- 背景采用浅暖色，降低纯白刺眼感
- 卡片白底承载信息，增强层次
- 状态色明确且固定（成功/警示/危险/信息）

建议色板（首版）：

- `--color-primary: #9D2933`
- `--color-primary-soft: #F6E9EA`
- `--color-bg: #F7F4F1`
- `--color-surface: #FFFFFF`
- `--color-text: #2B2B2B`
- `--color-text-secondary: #6B7280`
- `--color-border: #E7E2DD`
- `--color-success: #2F7D4A`
- `--color-warning: #B7791F`
- `--color-danger: #C53B3B`

### 3.3 字体与层级

- 采用系统可用中文优先字体栈，确保小程序与 H5 一致可读
- 标题层级清晰：页面标题 > 区块标题 > 列表标题 > 辅助信息
- 字重和字号遵循“信息优先”原则，避免过度装饰

### 3.4 形态与间距

- 圆角中等（卡片 16、按钮 12）
- 阴影轻量，仅用于建立层次
- 间距节奏统一为 8/12/16/24

## 4. CSS 架构设计

### 4.1 文件组织

在 `src/styles/` 下建立：

1. `tokens.scss`：颜色、字号、间距、圆角、阴影、层级
2. `mixins.scss`：少量复用规则（卡片、按钮态、文本截断）
3. `utilities.scss`：克制的通用工具类（间距、文字颜色）
4. `index.scss`：统一入口，聚合以上样式并注入全局

### 4.2 全局规则

- `App.vue` 仅保留页面底色、默认文字色、安全区相关基础样式
- 业务页面默认使用 scoped 样式，减少全局污染
- 所有业务颜色优先走 token，不直接写魔法值

## 5. 样式演示页（Style Showcase）

新增页面：`src/pages/style-showcase/index.vue`，用于集中查看与验收样式规范。

展示模块包括：

1. 色板（主色、中性色、状态色）
2. 文字层级（标题、正文、说明）
3. 按钮样式（主按钮、次按钮、警示按钮、禁用态）
4. 卡片样式（普通卡片、强调卡片）
5. 标签样式（通知标签、流程状态标签）
6. 列表项样式（通知条目、待办条目）
7. 表单控件外观基线（输入框、选择器容器）
8. 空状态示例

该页面用于“先看效果再落业务”，降低后续返工。

## 6. 响应式与平台兼容

- 以小程序移动端优先（常见 375 宽）
- 在 H5 下保证最小可读宽度与合理留白
- 避免依赖小程序专有组件样式特性，保持跨端一致

## 7. 验收标准

1. 全局 token 可被业务页面直接复用
2. Showcase 页可完整展示核心样式元素
3. 小程序端与 H5 端视觉一致性可接受
4. `npm run type-check` 和 `npm run build:h5` 可通过

## 8. 风险与控制

- 风险：红色使用过量导致视觉沉重
  - 控制：仅关键操作/状态使用主色，大面积区域使用中性色
- 风险：样式散落导致后续维护困难
  - 控制：统一 token 入口，禁止随意新增未命名语义色
- 风险：小程序与 H5 表现差异
  - 控制：先在 showcase 验证通用组件表现，再推广到业务页

## 9. 里程碑

1. 完成样式基础框架
2. 完成 showcase 页面
3. 运行并人工验收视觉基线
4. 再逐步迁移业务页面
