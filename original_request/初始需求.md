## 1.     项目概述

### 1.1 项目背景

目前学院学生事务（如党团流程、证明开具、政策咨询、就业通知等）主要依赖人工处理和微信群转发，存在重复性劳动多、信息滞后、学生对流程不熟悉等痛点。

### 1.2 目标用户

l  **终端用户（学生）：** 学院约 1000 名本科生及研究生。

l  **管理员（老师****/****助教）：** 团委老师、班主任及相关负责学生工作的助教。

### 1.3 核心目标

打造一个“一站式”服务窗口，实现事务办理电子化、政策咨询智能化、信息推送精准化。

## 2.     运行环境与技术要求

l  **产品形态：**用户端：方便终端用户浏览、操作（例如微信小程序），管理员端：方便管理员基于PC机进行统计分析，以及文档的导入导出等操作。

l  **数据库：** 人大金仓（Kingbase）数据库。

l  **身份验证：** 基于微信账号实名制，对接学生基本信息（学号、姓名等）。

l  **数据交互：** 由于无法直接对接校级“微人大”系统接口，需支持 Excel/Word/PDF 等文档的导出与批量导入。

## 3.     功能需求详述

### 模块一：智能问答与政策知识库

l  **智能咨询：** 录入学院/学校常用的政策文件（如奖助学金、休学复学、宿舍调整、查档调档、节假日放假等），通过关键词搜索或 AI 匹配，为学生提供即时回复。

l  **回复原则：** 优先基于标准答案或文件链接回复，避免大模型生成“幻觉”；敏感信息不直接展示，提供官方链接。

l  **模板下载：** 提供常用证明、请假条、活动预算表、简报等标准 Word/Excel 模板下载。

### 模块二：党团事务流程管理

l  **线性流程可视化：** 展示入党、入团的全流程（如：入党申请人 → 积极分子 → 发展对象 → 预备党员  → 正式党员）。

l  **进度追踪：** 学生可查询自己当前所处阶段及已完成的动作。

l  **理论自测（可选）：** 导入官方题库，支持学生进行党建知识自主测试。

### 模块三：信息集成与精准推送

l  **信息聚类：** 自动抓取或由管理员录入学校后勤、保卫处、就业办等部门的公众号通知。

l  **标签化分发：** 对信息打标签（如“就业”、“实习”、“计算机类”），根据学生画像（年级、专业、是否毕业生）进行精准推送。

l  **多渠道通知：** 支持通过小程序内消息、邮件或短信批量转发特定信息给目标群体。

## 4.     非功能需求

l  **安全性：** 学生身份证号、联系方式等基本信息需严格加密存储。

l  **并发性：** 预计并发量较低（日均同时在线约几十人）。

l  **可维护性：** 采用模块化开发，确保后续课程迭代时可方便地增加新功能（如出国申请、评价评优细则等）。

l  **可靠性：** 依托底层数据库保证数据的一致性和不丢失。
