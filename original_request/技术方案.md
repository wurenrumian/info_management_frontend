# 学院学生事务一站式系统 技术方案

## 1. 技术栈

| 层级   | 技术选型              | 说明                 |
| ---- | ----------------- | ------------------ |
| 后端   | Go + Gin          | 轻量高性能，方便部署         |
| ORM  | GORM              | 支持金仓 PostgreSQL 接口 |
| 数据库  | 人大金仓 (KingbaseES) | 使用 PostgreSQL 兼容接口 |
| 前端   | 微信小程序             | 手机端为主              |
| 文件存储 | 本地/OSS            | 文档库支持30M文件         |

## 2. 权限体系（四级）

| 级别  | 角色      | 权限说明                 |
| --- | ------- | -------------------- |
| 1   | 普通学生    | 查看、提交申请、使用知识库、查看个人信息 |
| 2   | 团支书/团干部 | 信息发布、审批、流程管理、本班事务    |
| 3   | 班主任/教师  | 查看本班/本年级数据、无审批权限     |
| 4   | 超级管理员   | 维护学生信息、系统配置、全部权限     |

## 3. 核心数据模型

### 3.1 用户表 (users)

| 字段名         | 类型                 | 描述                |
| ----------- | ------------------ | ----------------- |
| id          | bigint PK          | 主键                |
| student_id  | varchar(20) UNIQUE | 学号（唯一绑定）          |
| name        | varchar(50)        | 姓名                |
| openid      | varchar(100)       | 微信OpenID映射        |
| role        | int                | 权限级别 (1-4)        |
| class_id    | bigint FK          | 所属班级              |
| grade       | varchar(10)        | 年级                |
| major       | varchar(100)       | 专业                |
| extra_attrs | jsonb              | 杂项数据（休学/复学/特殊身份等） |
| created_at  | timestamp          | 创建时间              |
| updated_at  | timestamp          | 更新时间              |

**extra_attrs 示例结构：**
```json
{
  "status": "在读",        // 在读/休学/复学/毕业
  "special_identities": ["困难生", "少数民族"],
  "remarks": "因病休学"
}
```

### 3.2 班级表 (classes)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| class_name | varchar(50) | 班级名称（如"信院20级1班"） |
| grade | varchar(10) | 年级 |
| major | varchar(100) | 专业 |
| counselor_id | bigint FK | 班主任用户ID |
| head_student_id | bigint FK | 班长/团支书用户ID |
| created_at | timestamp | 创建时间 |

### 3.3 管理员表 (admins)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| user_id | bigint FK UNIQUE | 关联用户（一对一） |
| role | int | 管理员权限级别 |
| managed_class_ids | bigint[] | 管理的班级ID数组（班主任用） |
| created_at | timestamp | 创建时间 |
| last_login_at | timestamp | 最后登录时间 |

以上是**最基础**的数据结构

---
### 3.4 操作日志表 (admin_logs)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| admin_id | bigint FK | 操作用户 |
| action | varchar(50) | 操作类型 |
| target_type | varchar(30) | 目标实体类型 |
| target_id | bigint | 目标ID |
| detail | jsonb | 操作详情 |
| ip_address | varchar(50) | IP地址 |
| created_at | timestamp | 操作时间 |

### 3.5 党团阶段表 (party_progress)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| user_id | bigint FK | 学生用户 |
| type | varchar(10) | "party"入党/"league"入团 |
| stage | varchar(20) | 当前阶段 |
| stage_updated_at | timestamp | 阶段更新时间 |
| extra_info | jsonb | 扩展信息（思想汇报时间等） |

### 3.6 审批流程表 (approvals)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| applicant_id | bigint FK | 申请人 |
| type | varchar(30) | 审批类型（请假/盖章等） |
| status | varchar(20) | pending/approved/rejected/withdrawn |
| form_data | jsonb | 表单数据 |
| current_approver_id | bigint FK | 当前审批人 |
| history | jsonb | 审批历史记录 |
| semester | varchar(10) | 所属学期 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

### 3.7 信息发布表 (announcements)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| title | varchar(200) | 标题 |
| content | text | 正文 |
| author_id | bigint FK | 作者 |
| tags | varchar[] | 标签数组 |
| target_grades | varchar[] | 目标年级 |
| target_majors | varchar[] | 目标专业 |
| attachments | jsonb | 附件列表（含路径、大小） |
| published_at | timestamp | 发布时间 |

### 3.8 知识库表 (knowledge_base)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| question | text | 问题 |
| answer | text | 标准答案 |
| keywords | varchar[] | 关键词数组 |
| attachments | jsonb | 附件链接 |
| created_by | bigint FK | 创建人 |
| updated_at | timestamp | 更新时间 |

### 3.9 文档库表 (documents)

| 字段名 | 类型 | 描述 |
|--------|------|------|
| id | bigint PK | 主键 |
| title | varchar(200) | 文档名 |
| file_path | varchar(500) | 存储路径 |
| file_size | bigint | 文件大小 |
| uploader_id | bigint FK | 上传人 |
| category | varchar(50) | 分类 |
| created_at | timestamp | 上传时间 |

## 4. API 设计原则

- 统一 RESTful 风格
- 认证：JWT + 学号绑定
- 权限校验在中间件层完成
- 文件上传建议分片（30M限制）

## 5. 模块划分

由于各模块耦合度低，可分阶段独立开发：

1. **用户与权限模块**（基础）
2. **班级管理模块**（基础）
3. **党团流程模块**
4. **知识库与问答模块**
5. **信息发布模块**
6. **审批流程模块**
7. **文档库模块**
